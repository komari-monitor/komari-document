# 其他常见问题

## 内存显示与 xxx 不一致

自 Agent 1.1.40 版本开始，Komari 的内存统计逻辑与 htop 一致。
这意味着内存使用率的计算方式发生了变化，可能会导致与之前版本或其他监控工具显示的数据不一致。
具体来说，Komari 现在将缓存和缓冲区内存视为可用内存的一部分，从而提供更准确的内存使用情况。

## 磁盘统计不正确

Komari 使用简单的挂载点+文件系统类型判断逻辑来过滤磁盘。如果您的系统中存在一些特殊的挂载点或文件系统类型，可能会导致磁盘统计不正确。
您可以通过 Agent 的 `--include-mountpoint` 和 `--exclude-mountpoint` 参数来手动指定需要监控或排除的挂载点。

## 如何配置按月重置网络流量

在安装时勾选参数 `网络统计月重置日` ，并设置为您运营商的计费重置日，即可启用按月重置网络流量统计功能。

## 如何打开流量进度条

在后台中，点击服务器列表中的铅笔图标，进入服务器编辑页面，下拉找到 `流量阈值` 选项，设置为您希望的流量阈值，保存后即可看到流量进度条。

注：请不要超过 BIGINT 最大值（`9,223,372,036,854,775,807` 字节，约 9 PB），否则可能会导致显示异常。
